## 2023-10-18 - [Testing Improvement for History Logger]
**Learning:** When using `loadBrowserScript` to test modules that expect global objects mapped to `self` or `window` (like `root.ExtensityStorage`), passing the mock directly in the `globals` argument only places it on the sandbox root (Node's global context), not necessarily on `self`. To ensure the browser-emulated `root` handles it, you must explicitly pass `self: { ExtensityStorage: mockStorage }` in the globals.
**Action:** Always inject explicitly mapped global mock objects into `self` when testing legacy browser scripts using `loadBrowserScript` to prevent ReferenceErrors and ensure compatibility.
